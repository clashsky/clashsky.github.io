你可以把 Clash 小火箭想象成一个“网络交通指挥官”。它不像传统 VPN 那样将所有流量都一股脑地导入一个隧道，而是根据你设定的规则，精准地判断哪些流量需要走代理（比如访问被屏蔽的国外网站），哪些流量直连（比如访问国内网站或局域网），哪些流量需要被拦截（比如广告或恶意域名）。这种“智能分流”机制，不仅极大地提高了网络访问速度，还能避免不必要的带宽浪费，让你的网络资源得到最优化利用。

二、Clash 小火箭的安装与部署 —— 从零开始的详细指南

对于初学者来说，安装和配置 Clash 小火箭可能是最大的门槛。但实际上，只要按照正确的步骤操作，整个过程并不复杂。下面，我们以 Windows 和 macOS 平台为例，为你提供一份清晰的安装指南。

2.1 准备工作：获取核心与客户端

首先，你需要下载两样东西：Clash 核心 和 Clash 小火箭客户端。

  • Clash 核心：这是 Clash 小火箭运行的基础引擎。你可以访问 Clash 的官方 GitHub 仓库（搜索“Clash Core”），根据你的操作系统选择对应的版本下载。例如，Windows 64位系统选择 clash-windows-amd64.zip，macOS Intel 芯片选择 clash-darwin-amd64.zip，Apple Silicon 芯片选择 clash-darwin-arm64.zip。下载后解压到一个你容易找到的文件夹（例如 C:\Clash 或 ~/Clash）。
  • Clash 小火箭客户端：这是图形化界面程序。你可以在其官方网站或各大应用商店（如 App Store、Google Play）搜索“Clash 小火箭”下载。注意区分不同平台的版本，例如 Windows 版通常是 .exe 或 .msi 安装包，macOS 版是 .dmg 文件。

2.2 安装步骤详解

Windows 平台：

  1. 安装 Clash 核心：将下载的 clash-windows-amd64.zip 解压到一个文件夹（例如 C:\Clash）。为了后续方便，建议将该文件夹路径添加到系统环境变量 PATH 中，这样你就可以在命令行中直接运行 clash 命令了（非必需，但推荐）。
  2. 安装 Clash 小火箭客户端：双击下载的 .exe 或 .msi 安装包，按照提示完成安装。安装完成后，桌面会出现一个火箭图标。
  3. 首次启动配置：双击桌面图标启动 Clash 小火箭。首次运行时，程序会提示你选择 Clash 核心文件的位置。点击“浏览”，找到你刚才解压的 clash.exe 文件并选中。然后，程序会要求你导入配置文件或订阅链接。

macOS 平台：

  1. 安装 Clash 核心：将下载的 clash-darwin-amd64.zip 或 clash-darwin-arm64.zip 解压，将解压后的 clash 可执行文件移动到 /usr/local/bin/ 目录下（需要输入管理员密码）。你也可以将它放在任意位置，但为了方便，建议放在系统 PATH 路径下。
  2. 安装 Clash 小火箭客户端：打开下载的 .dmg 文件，将 Clash 小火箭图标拖拽到“应用程序”文件夹中。首次打开时，系统可能会提示“无法验证开发者”，你需要前往“系统偏好设置” -> “安全性与隐私” -> “通用”，点击“仍要打开”。
  3. 首次启动配置：启动 Clash 小火箭，同样会要求你指定 Clash 核心文件。点击“选择核心”，定位到 /usr/local/bin/clash 文件。然后，进入配置导入环节。

2.3 导入订阅与配置文件

这是最关键的一步。Clash 小火箭支持两种方式导入配置：

  • 订阅链接：这是最常用的方式。你需要从你的代理服务商那里获取一个订阅链接（通常是一串以 https:// 开头的 URL）。在 Clash 小火箭的“配置”或“订阅”页面，点击“添加”或“导入”，粘贴该链接，程序会自动下载并解析配置文件，其中包含了所有可用的节点信息。
  • 本地配置文件：如果你有现成的 YAML 格式配置文件（例如从 Clash 官方示例或社区分享中获得），也可以直接导入。点击“导入文件”，选择你的 .yaml 文件即可。

导入成功后，你会在“节点”或“代理”列表中看到所有可用的服务器节点。每个节点通常会显示名称、类型（如 Vmess、Shadowsocks）、延迟（Ping 值）等信息。

2.4 启动与连接

  1. 选择节点：在节点列表中，选择一个延迟较低、速度较快的节点。你可以点击节点名称旁边的“测试延迟”按钮，查看实时响应时间。建议选择延迟在 200ms 以下的节点，以获得较好的上网体验。
  2. 开启系统代理：在 Clash 小火箭的主界面，通常有一个“开关”按钮。点击它，程序会启动 Clash 核心，并自动修改你的系统代理设置。此时，你的所有网络流量都会经过 Clash 小火箭的智能分流处理。
  3. 验证连接：打开浏览器，访问一个被屏蔽的网站（如 Google、YouTube、Twitter 等），如果能够正常打开，说明配置成功。你也可以访问 ip.sb 或 whatismyipaddress.com 等网站，查看你的 IP 地址是否已经变为代理节点的 IP。

三、Clash 小火箭的深度配置 —— 打造专属网络规则

Clash 小火箭的强大之处在于其高度可定制的规则系统。通过合理配置规则，你可以实现“国内流量直连，国外流量代理，广告流量拦截”等精细化控制，让网络体验更加丝滑。下面，我们来详细介绍几个核心配置项。

3.1 策略组与节点选择

在 Clash 小火箭的“策略组”设置中，你可以创建不同的策略组，并将节点分配到不同的组中。常见的策略组包括：

  • Proxy：用于存放所有代理节点。你可以在这里设置“自动选择”或“故障转移”等模式。自动选择会根据节点的延迟和丢包率动态切换最优节点；故障转移则会在当前节点不可用时自动切换到下一个节点。
  • Direct：用于存放直连节点。通常不需要手动配置，系统默认会有一个“直连”策略组，用于处理国内流量。
  • Reject：用于存放需要拦截的域名或 IP，例如广告、追踪器、恶意网站等。

你还可以根据自己的需求创建自定义策略组，例如“游戏组”专门用于游戏加速，“流媒体组”用于解锁 Netflix、Disney+ 等区域限制。

3.2 规则配置详解

规则是 Clash 小火箭的灵魂。它告诉 Clash 核心：当访问某个域名或 IP 时，应该使用哪个策略组。规则由“类型”、“匹配内容”和“策略”三部分组成。

常见的规则类型包括：

  • DOMAIN-SUFFIX：匹配域名后缀，例如 DOMAIN-SUFFIX,google.com,Proxy 表示所有以 google.com 结尾的域名（如 www.google.com、mail.google.com）都走 Proxy 策略组。
  • DOMAIN-KEYWORD：匹配域名关键词，例如 DOMAIN-KEYWORD,youtube,Proxy 表示所有包含 youtube 的域名都走 Proxy 策略组。
  • DOMAIN：精确匹配域名，例如 DOMAIN,www.baidu.com,Direct 表示 www.baidu.com 直连。
  • IP-CIDR：匹配 IP 地址段，例如 IP-CIDR,192.168.0.0/16,Direct 表示局域网 IP 直连；IP-CIDR,10.0.0.0/8,Direct 表示私有 IP 直连。
  • GEOIP：根据 IP 的地理位置匹配，例如 GEOIP,CN,Direct 表示所有中国 IP 直连；GEOIP,US,Proxy 表示美国 IP 走代理。
  • FINAL：最终规则，如果所有前面的规则都没有匹配到，则使用此规则。通常设置为 FINAL,Proxy 或 FINAL,Direct，取决于你的需求。

一个典型的规则配置顺序建议：

  1. 广告拦截规则（Reject）：将已知的广告域名、追踪器域名匹配到 Reject 策略组。
  2. 国内域名规则（Direct）：将常用的国内网站（如百度、淘宝、腾讯等）匹配到 Direct 策略组。
  3. 国外域名规则（Proxy）：将被屏蔽的国外网站（如 Google、YouTube、Facebook 等）匹配到 Proxy 策略组。
  4. 国内 IP 规则（Direct）：将中国 IP 地址段匹配到 Direct 策略组。
  5. 最终规则（FINAL）：通常设置为 FINAL,Proxy，确保所有未匹配的流量都走代理，避免漏网之鱼。

Clash 小火箭通常内置了丰富的规则集，你可以在“规则”页面查看和编辑。你也可以从社区下载更完善的规则集（如“神机规则”、“ACL4SSR”等），直接导入使用。

3.3 进阶功能：DNS 与 TUN 模式

  • DNS 配置：Clash 小火箭支持自定义 DNS 服务器。建议使用 https://dns.alidns.com/dns-query（阿里 DNS）或 https://dns.quad9.net/dns-query（Quad9 DNS）等支持 DoH（DNS over HTTPS）的服务器，可以防止 DNS 污染，提升解析速度和安全性。
  • TUN 模式：这是 Clash 小火箭最强大的功能之一。开启 TUN 模式后，Clash 会在系统层面创建一个虚拟网卡，所有网络流量（包括非 HTTP/S 流量，如游戏、SSH、VPN 等）都会被强制经过 Clash 处理。这意味着你无需为每个应用程序单独配置代理，所有软件都能自动享受加速。对于游戏玩家和需要全局代理的用户来说，TUN 模式是必备功能。需要注意的是，开启 TUN 模式可能会与某些杀毒软件或防火墙冲突，需要根据实际情况调整。

FAQ

Hiddify Next 如何为不同用户设置流量限制？
管理员可为每个用户设置独立带宽限制和流量配额。超过限制时，可自动降速或阻止访问，确保服务器资源合理分配，避免单用户占用过多带宽，提升整体网络效率和公平性。
Trojan 的连接失败常见原因有哪些？
Trojan 连接失败可能由于伪装域名错误、TLS 证书过期、防火墙阻挡端口或服务器不可达。通过查看客户端日志可以定位具体原因，并对症下药解决问题。
Sing-box 是什么？
Sing-box 是新一代多协议代理框架，支持 VMess、VLess、Trojan、Hysteria、ShadowTLS 等协议。它比传统 V2Ray 更轻量化、模块化，并针对移动端和 Linux 系统进行了性能优化，受到开发者和高级用户的欢迎。
VMess AEAD 加密在高封锁地区的优势？
AEAD 加密可防止数据篡改和流量特征泄露，降低被 DPI 检测风险。在高封锁地区使用，可保证节点长期稳定可用，同时提升数据传输安全性和访问可靠性。
SSR 的协议和混淆组合如何选择最适合的？
选择模拟 HTTPS 或 HTTP 的混淆方式，再搭配 AEAD 或协议插件，可以隐藏流量特征并提高抗封锁能力。不同网络环境需测试，选择最稳定和可靠的组合。
Shadowrocket 支持哪几种代理协议？
Shadowrocket 支持多种协议，包括 Shadowsocks、Vmess、VLess、Trojan、HTTP、SOCKS5 等。它还可解析部分 QuantumultX 与 Clash 的订阅格式，适合 iOS 用户进行全协议代理管理与切换。
QuantumultX 如何通过策略组优化多应用分流？
QuantumultX 可以根据应用类型、端口和协议设置策略组，自动选择最优节点处理不同流量。这样可保证网页浏览、视频播放和游戏访问顺畅，提高整体网络效率。
Clash 如何实现分应用代理？
Clash 可通过策略组和规则文件按应用分流，将特定应用的流量转发到指定节点。例如游戏走低延迟节点，网页浏览走普通节点，实现高效分流和优化网络速度。
SSR 的混淆方式选择有什么讲究？
混淆方式决定流量伪装特性，不同方式适应不同封锁环境。常用方式如 tls1.2_ticket_auth 模拟 HTTPS，http_simple 模拟 HTTP 请求，根据环境选择可提高抗封锁能力。
VMess 节点的 alterId 参数还需要设置吗？
在新版 V2Ray/Xray 内核中，alterId 已被废弃，不再影响认证逻辑。建议用户设置为 0 或删除该字段，以避免连接异常或不兼容情况。
